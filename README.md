# 1С Ассистент — Telegram-бот для онбординга сотрудников

Телеграм-бот, который помогает новым сотрудникам быстрее освоить работу в 1С (ERP). Вместо того чтобы отвлекать руководителя или листать длинные инструкции, сотрудник просто пишет вопрос в чат — и получает конкретный ответ.

---

## Зачем это нужно

Онбординг в 1С — это стресс. Новый сотрудник не помнит алгоритмы, боится ошибиться, постоянно дёргает коллег. Руководитель теряет время на одни и те же вопросы.

Этот бот решает проблему просто: база знаний компании всегда под рукой, в Telegram, в любой момент.

**Что получает сотрудник:**
- Быстрый ответ на вопрос в свободной формулировке
- Пошаговые инструкции по работе в 1С
- Уверенность в сложных ситуациях

**Что получает руководитель:**
- Меньше отвлечений на базовые вопросы
- Новые сотрудники выходят на самостоятельную работу быстрее

---

## Как это работает

Бот использует технологию RAG (Retrieval-Augmented Generation): когда сотрудник задаёт вопрос, бот ищет релевантные фрагменты из базы знаний и передаёт их языковой модели, которая формулирует чёткий ответ.

```
Сотрудник пишет вопрос
        ↓
Бот ищет нужный фрагмент в базе знаний (векторный поиск, ChromaDB)
        ↓
Языковая модель формирует ответ на основе найденного (OpenRouter)
        ↓
Сотрудник получает конкретную инструкцию
```

### Обработка PDF с картинками

Инструкции по 1С содержат скриншоты интерфейса, которые несут важный контекст. Для их обработки реализован отдельный пайплайн:

```
PDF-файл
  ├── Текст → извлекается через PyMuPDF
  └── Картинки → анализируются Vision LLM (Gemma 3 12B через Ollama)
          ↓
Текст и описания картинок склеиваются в порядке расположения на странице
          ↓
Сохраняется в parsed_docs/ для проверки
          ↓
Загружается в ChromaDB вместе с остальными документами
```

## Технологии

| Компонент | Инструмент |
|---|---|
| Telegram-бот | aiogram 3 |
| Языковая модель | OpenRouter (цепочка из нескольких free-моделей с fallback) |
| Vision LLM (анализ картинок) | Gemma 3 12B через Ollama (локально) |
| Векторная база знаний | ChromaDB |
| RAG-пайплайн | LangChain |
| Эмбеддинги | sentence-transformers (all-MiniLM-L6-v2) |
| LLM-клиент | openai SDK (с кастомным base_url) |
| Извлечение текста из PDF | PyMuPDF (fitz) |
| Язык | Python 3.11+ |

---

## Требования к запуску

- Python 3.11+
- Telegram Bot Token (получить у @BotFather)
- API ключ OpenRouter (бесплатно на openrouter.ai)
- [Ollama](https://ollama.com) с моделью `gemma3:12b` — для обработки картинок из PDF
- ~300 МБ свободного места на диске (+ ~8 ГБ для модели Ollama)

## Установка

```bash
git clone https://github.com/ваш-username/1c-assistant-bot
cd 1c-onboarding-bot
pip install -r requirements.txt
```

Создайте файл `.env`:

```env
TELEGRAM_TOKEN=ваш_токен
OPENROUTER_API_KEY=ваш_ключ
```

### Загрузка базы знаний

**Шаг 1.** Если в инструкциях есть PDF с картинками — обработайте их через Vision LLM:

```bash
# Все PDF из knowledge_base/
python parse_pdf_with_vision.py

# Или конкретный файл
python parse_pdf_with_vision.py --file knowledge_base/manual.pdf
```

Результат сохранится в `parsed_docs/` — можно проверить качество перед загрузкой.

**Шаг 2.** Загрузите все документы в ChromaDB (PDF, DOCX из `knowledge_base/` + TXT из `parsed_docs/`):

```bash
python load_documents.py
```

**Шаг 3.** Запустите бота:

```bash
python main.py
```

## Структура проекта

```

├── bot/
│   ├── __init__.py
│   ├── handlers.py              # Обработка сообщений
│   ├── keyboards.py             # Меню и кнопки
│   └── logger.py                # Настройка логирования
├── rag/
│   ├── __init__.py
│   ├── retriever.py             # Поиск по базе знаний (ChromaDB)
│   └── llm.py                   # Запросы к языковой модели (OpenRouter)
├── parse_pdf_with_vision.py     # Парсер PDF: текст + описание картинок через Vision LLM
├── test_vision.py               # Тест Vision LLM на одной картинке
├── load_documents.py            # Загрузка документов в ChromaDB
├── knowledge_base/              # Исходные документы (PDF, DOCX) — не коммитятся
│   └── .gitkeep
├── parsed_docs/                 # TXT-результаты парсинга PDF — не коммитятся
├── logs/                        # Логи бота — не коммитятся
├── chroma_db/                   # Векторная БД — не коммитятся
├── .env                         # Токены (не коммитить!)
├── .gitignore
└── requirements.txt
```
